🚀 How It Works

This app uses WebRTC — the same technology used by Google Meet and WhatsApp video calls — to stream live video directly from sender phones to your receiver screen, peer-to-peer over the internet. A small "signaling server" is needed to introduce devices to each other (but video goes directly between devices for lowest latency).

⚡ Quickest Free Setup (PeerJS Cloud)

1
No server needed for testing! This app uses PeerJS's free public cloud server by default. Just open this HTML file in a browser on any device.
2
On your receiver phone (Phone B): Open this file in Chrome, tap RECEIVER tab. Leave it open.
3
On your sender phone (Phone A): Open this file in Chrome, tap SENDER tab, then tap ▶ START CAMERA.
4
When Chrome asks for camera permission — tap Allow. The stream connects automatically. No codes needed.
5
On the receiver screen, the camera feed appears automatically within seconds. You can switch front/back camera using the button on each feed.
⚠ The PeerJS free server may be slow or unavailable at times. For reliable 24/7 use, host your own signaling server (see below).

🌐 Host Your Own Signaling Server (Recommended)

1
Get a free server: Render.com, Railway.app, or Glitch.com all offer free Node.js hosting.
2
Create a new Node.js project and paste this server.js:

const { PeerServer } = require('peer'); const server = PeerServer({ port: process.env.PORT || 9000, path: '/livecam' }); console.log('Signaling server running');
3
Add package.json:

{"name":"livecam","scripts":{"start":"node server.js"},"dependencies":{"peer":"^0.7.0"}}
4
Deploy to Render/Railway. Get your URL like https://mylivecam.onrender.com.
5
In the SENDER tab, enter your server URL in the input box and tap SET. Done!

📱 How to Share This App on Internet

1
Upload livecam.html to Netlify Drop (netlify.com/drop) — drag & drop, get a public URL in 30 seconds. Free forever.
2
Or use GitHub Pages: Create a repo, upload the file, enable Pages. Get a URL like yourusername.github.io/livecam.
3
Open the URL on all your phones. HTTPS is required for camera access — both Netlify and GitHub Pages give HTTPS free.

⚠ Important Notes & Limitations

HTTPS Required: Camera permission only works on HTTPS pages (or localhost). Must host on Netlify/GitHub Pages — not local file://

Same Page on All Devices: Every device (sender and receiver) must open the same URL.

Bandwidth: Each camera uses ~1-3 Mbps. 10 cameras = 10-30 Mbps on the receiver's connection.

Battery Drain: Streaming video drains battery quickly on sender phones. Keep them plugged in.

Mobile Chrome: Use Chrome on Android. Safari on iOS may have limitations.

Background: On mobile, browsers may pause when the screen locks. Keep screens on while streaming.

PeerJS Free Limit: PeerJS cloud has connection limits. Self-host for reliable use.
