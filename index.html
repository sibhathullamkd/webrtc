🔧 How It Works

Uses WebRTC (direct peer-to-peer video) via PeerJS. The receiver holds a daily-rotating peer ID slot. Senders find it automatically — no codes. Works over internet from any device on any network including 4G/5G.

⚡ Same Device (2 Chrome tabs)

1
Open the URL in Chrome → Tab 1 → stays on RECEIVER tab.
2
Open a new Chrome tab → same URL → SENDER tab → tap ▶ START CAMERA → allow permission.
3
Switch to Tab 1. Camera appears within ~3 seconds. ✅
⚠ If stuck: on Receiver tab tap ↺ RESTART, then on Sender tab tap ■ STOP then ▶ START CAMERA again.

📱 Different Devices

1
Receiver (Phone B): Open URL in Chrome → stay on RECEIVER tab → leave it open.
2
Sender (Phone A): Open same URL → SENDER tab → START CAMERA → allow camera.
3
Video appears on Phone B. Repeat for more cameras — each gets its own tile.
4
Tap ⟳ FLIP on any tile to switch that camera front/back.

⚠ Important Notes

Keep screen ON on sender phones — Chrome pauses camera when screen locks.

Use Chrome on Android for best support.

HTTPS required — GitHub Pages provides this automatically.

If no connection after 15s: tap ↺ RESTART on receiver, stop/restart sender.

Bandwidth: each camera ≈ 1–2 Mbps. 10 cameras = 10–20 Mbps on receiver.
