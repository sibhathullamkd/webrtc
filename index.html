🔧 How It Works

Dpwed-webRTC cam uses WebRTC peer-to-peer technology. The receiver holds a daily peer ID slot. Senders find it automatically via localStorage (same device) or slot-probing (different devices). Camera + microphone stream together. Audio is muted by default on receiver — tap 🎤 on any tile to unmute.

⚡ Same Device — 2 Chrome Tabs

1
Open your GitHub Pages URL in Chrome → Tab 1 stays on RECEIVER.
2
Open a new tab → same URL → tap SENDER tab.
3
Tap ▶ START CAMERA + MIC → allow both camera AND microphone.
4
Switch to Tab 1. Camera feed appears within ~3 seconds. ✅
⚠ If stuck after 15s: on Receiver tap ↺ RESTART RECEIVER, then on Sender tap ■ STOP → ▶ START.

📱 Different Devices (10+ Cameras)

1
Receiver phone: Open URL → stay on RECEIVER tab → keep screen on.
2
Each sender: Open same URL → SENDER tab → START CAMERA + MIC → allow both permissions.
3
Each sender appears as its own tile on the receiver screen automatically.
4
Tap ⟳ FLIP on a tile to switch that camera front/back. Tap 🎤 to unmute/mute audio.

⚠ Important Notes

Keep screen ON on sender — Chrome pauses camera when screen locks.

Chrome on Android — best WebRTC support. Firefox also works.

HTTPS required — GitHub Pages provides this free.

Bandwidth: each camera+audio stream ≈ 1.5–2.5 Mbps. Plan accordingly.

Battery: streaming drains battery fast on senders. Keep them plugged in.
