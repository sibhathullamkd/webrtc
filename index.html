🔥 Firebase Setup One-time · Free

Uses Firebase Realtime Database as signaling relay — free, reliable, no conflicts. Video goes direct peer-to-peer so it's fast.
1
Go to console.firebase.google.com → open your existing project.
2
Build → Realtime Database → Create Database → Start in test mode → Enable
3
Copy the URL: https://your-project-default-rtdb.firebaseio.com
4
Paste it below and tap Save. Do this on every device once.
No URL saved yet.
⚠ Same URL on all devices. In Firebase → Realtime Database → Rules: set .read and .write to true → Publish.

⚡ How to Use

1
Receiver: Open URL in Chrome → stay on Receiver tab → green SIGNAL dot = ready.
2
Sender: Open same URL → Sender tab → Start Camera + Mic → allow both.
3
Feed appears on receiver in ~3 seconds. Add as many senders as you want.
4
Tap ↻ Flip on any tile to switch camera. Tap 🔇 to unmute audio.

⚠ Notes

Keep screen ON on sender — Chrome pauses camera when screen locks.

Use Chrome on Android for best WebRTC compatibility.

HTTPS required — GitHub Pages provides this automatically.

Bandwidth: ~1.5–2.5 Mbps per camera. 10 cameras ≈ 20 Mbps on receiver.

If stuck: tap ↺ Restart on receiver, then Stop → Start on sender.
